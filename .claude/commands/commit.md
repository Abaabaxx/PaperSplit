请按以下步骤操作，**不要急着提交**：

1. 运行 `git status` 查看当前变更文件列表
2. 运行 `git diff` 和 `git diff --staged` 查看具体改动内容
3. 根据改动内容，用**中文**拟一条简洁的提交信息（格式：`<类型>：<描述>`，类型如：新增、修复、重构、文档、配置等）
4. 将以下内容整理后展示给我：
   - 待提交的文件列表
   - 拟好的提交信息
5. **停下，等待我确认。** 在我明确说"确认"或"提交"之前，不要执行任何 git 操作。

确认后，执行：
- `git add` 相关文件
- `git commit -m "<拟好的提交信息>"`
